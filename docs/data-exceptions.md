# Data Exceptions

此文件記錄中選會已揭露資料本身的缺漏, 錯誤, 或需要人工判讀的例外處理摘要.

不記錄爬蟲策略, API 結構, 網頁解析方式, 或可自動化取得資料的技術細節. 這些內容應放在 `cec-scraper` skill.

## 記錄範圍

此文件適合記錄:

- 原始資料缺欄位, 缺選舉, 缺候選人, 或內容空白.
- 原始資料有明顯登打錯誤, 需要人工補正或合併.
- 同一筆歷史資料在不同來源互相矛盾, 需要保留判讀摘要.
- 因原始資料品質問題而對 `_data/`, DB, 或 mapping 採取的人工處理.

此文件不適合記錄:

- 中選會靜態 API URL 公式.
- subject_id, theme_id, data_level 等爬蟲規格.
- HTML 或 JSON 結構變化.
- 可透過調整爬蟲穩定解決的問題.

## 新增例外格式

新增資料例外時, 優先補齊以下資訊:

- 選舉: 年份與 election type.
- 地區: 官方地區名稱, 使用 `臺` 不寫作 `台`.
- 資料問題: 原始資料缺漏或錯誤的最小描述.
- 影響範圍: 影響哪些候選人, 選區, 欄位, 或輸出資料.
- 處理摘要: 本專案實際採取的人工處理.
- 補正來源: 若人工補資料, 記錄來源 URL, 文件, 或判讀依據.
- 驗證方式: 如何確認目前資料已符合處理摘要.
