候選人 Merge UI — 三種方案示意

情境：candidates.yaml 已存在（含 mayor 2022 資料），現在執行 president 2024

方案 A  —  終端機逐筆確認（每筆都問）
$ uv run python main.py --type president --year 2024
解析 第16任總統副總統選舉.xlsx ...
比對現有 candidates.yaml ...
NEW  吳欣盈  birthday:1978 | 國家元首 2024 | 台灣民眾黨 | elected:0
→ 現有 yaml 找不到此人
新增？ [y] 是  [s] 跳過  [e] 編輯後新增 > y
EXISTS  柯文哲  birthday:1959 | 國家元首 2024 | 台灣民眾黨 | elected:0
→ 現有 id_柯文哲（已有 2014/2018 縣市首長），將新增一筆 election
合併？ [y] 是  [s] 跳過  [e] 編輯後合併 > _
CONFLICT  許淑華  birthday:1971 | 國家元首 2024 | 民主進步黨 | elected:0
→ yaml 中有同名 2 人：
[1] id_許淑華_1973  臺北市議員 2016 (民主進步黨)
[2] id_許淑華_1975  南投縣長 2014 (中國國民黨)
歸屬？ [1][2] 合併至對應人  [n] 新增第三人  [s] 跳過 >
進度 3 / 6 筆
✅ 流程透明，每筆都能控制    ⚠️ 資料量大時很繁瑣（6筆總統候選人還好，但縣市議員可能幾千筆）
方案 B  —  輸出草稿 YAML，手動用編輯器合併
$ uv run python main.py --type president --year 2024
解析完畢，輸出草稿至 draft_president_2024.yaml
請用編輯器審閱後手動合併至 candidates.yaml
統計：
NEW 2 筆（可直接貼入）
MERGE 3 筆（需補 election 至現有條目）
CONFLICT 1 筆（需人工判斷同名歸屬）
draft_president_2024.yaml
# ── NEW ──────────────────────────────────────
# 現有 yaml 找不到此人，可直接複製貼入
- name: 吳欣盈
id: id_吳欣盈
birthday: 1978
elections:
- year: 2024
type: 國家元首
region: null
party: 台灣民眾黨
elected: 0
# ── MERGE ────────────────────────────────────
# 請補至 candidates.yaml 中 id_柯文哲 的 elections 下
- year: 2024
type: 國家元首
region: null
party: 台灣民眾黨
elected: 0
# ── CONFLICT ─────────────────────────────────
# 同名 2 人，請判斷是哪一位，或新增第三人
# 現有: id_許淑華_1973 (臺北市議員), id_許淑華_1975 (南投縣長)
- name: 許淑華 # ← 請確認
birthday: 1971
elections:
- year: 2024
type: 國家元首
party: 民主進步黨
elected: 0
✅ 可以慢慢看、來回翻    ⚠️ 合併動作完全手動，MERGE 那段需要自己找到對應位置貼入
方案 C  —  自動處理無衝突，只暫停在需要判斷的地方（推薦）
$ uv run python main.py --type president --year 2024
解析 第16任總統副總統選舉.xlsx ...
